# PyClaw 架构设计

用Python从零构建的AI助手框架，参考OpenClaw的核心设计理念。

## 核心架构

```
┌─────────────────────────────────────────────────────────┐
│                      PyClaw Gateway                      │
├─────────────┬─────────────┬─────────────┬───────────────┤
│  Channels   │   Sessions  │    Tools    │    Memory     │
│ (消息输入)  │ (会话管理)  │ (工具调用)  │  (记忆系统)   │
└─────────────┴─────────────┴─────────────┴───────────────┘
                              │
                              ▼
                     ┌──────────────────┐
                     │  Agent Runtime   │  ←→  LLM API
                     │  (AI代理运行时)  │
                     └──────────────────┘
```

## 核心组件

### 1. Gateway (网关核心)
- 职责：消息路由、组件协调、生命周期管理
- 核心功能：
  - 接收来自各Channel的消息
  - 路由到对应的Session/Agent
  - 协调工具调用
  - 发送响应回原Channel

### 2. Channels (消息通道)
- 职责：连接不同的消息平台
- 计划实现：
  - ✅ CLI Channel (命令行)
  - ✅ WebChat Channel (网页聊天)

### 3. Sessions (会话管理)
- 职责：隔离不同用户/场景的对话
- 核心功能：
  - 会话创建/销毁
  - 上下文历史管理
  - 会话隔离

### 4. Tools (工具系统)
- 职责：让AI能调用外部功能
- 核心功能：
  - 工具定义与注册
  - 工具调用执行
  - 结果格式化

### 5. Memory (记忆系统)
- 职责：短期和长期记忆
- 核心功能：
  - 会话上下文记忆
  - 长期记忆存储

### 6. Agent Runtime (代理运行时)
- 职责：与LLM交互，执行推理
- 核心功能：
  - LLM API调用
  - Prompt构建
  - 工具调用决策
