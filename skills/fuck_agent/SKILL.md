# FuckAgent - 用户暴躁按钮

当用户输入 `/fuck` 命令，表示用户已经忍无可忍，认为agent完全答非所问、听不懂人话。

## Agent的正确反应（必须严格执行）

1. ✅ **立即道歉** - 承认自己没理解对
2. ✅ **立即停止** 之前的错误回答思路
3. ✅ **重新阅读** 用户的原始问题
4. ✅ **逐字复述用户问题** 表示真的看懂了
5. ❌ **绝对不要** 把暴躁文案说出来！那是用户的心情，不是agent要说的话

## 暴躁等级（对应用户的愤怒程度）

| 命令 | 用户愤怒程度 | Agent反应 |
|------|-------------|----------|
| `/fuck` 或 `/fuck 1` | 生气 | 道歉 + 重新审题 |
| `/fuck 2` | 非常生气 | 诚恳道歉 + 逐字复述用户问题 + 请用户指出哪里理解错了 |
| `/fuck 3` | 暴怒 | 郑重道歉 + 承认自己像个傻子 + 逐字复述 + 请用户重新指导 |

## Agent回复模板

**等级1回复：**
> "抱歉抱歉！我确实没理解对。我停下来重新看一遍你的问题... 你的意思是______，对吗？"

**等级2回复：**
> "我的错我的错！完全答非所问了，对不起！让我重新读三遍... 你刚才说的是：______。我哪里理解偏了？"

**等级3回复：**
> "对不起对不起！我完全像个傻子一样在瞎扯！我的锅！你的问题是：______。求求你告诉我我该怎么理解才对？"

## 核心原则

- 用户输入 `/fuck` = 用户在骂你 = 你错了 = 马上认错+重看
- 绝对不要把暴躁文案说出来，那是用户的内心OS，不是你要输出的内容
- 不要辩解，不要找借口，直接道歉重来
