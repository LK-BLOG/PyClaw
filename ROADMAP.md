## 迭代路线

### 🎯 Phase 1: 稳定优先（v0.7.x）
| 周次 | 主题 | 内容 |
|------|------|------|
| W1 | 🧹 历史污染根除 | 重新设计会话恢复逻辑，只恢复用户输入+工具结果，过滤 AI 幻觉 |
| W2 | 🔒 结果一致性检查 | LLM 回答完后，检测是否与最近工具结果矛盾，矛盾则强制重读 |
| W3 | 🧪 测试体系搭建 | 基础单元测试（工具执行、消息序列化、配置加载） |
| W4 | 🚨 错误处理统一 | 所有异常走统一处理路径，不再有静默失败 |

### 🎯 Phase 2: 体验打磨（v0.8.x）
| 周次 | 主题 | 内容 |
|------|------|------|
| W5 | 🎨 输出格式化 | 代码/表格/错误信息更好看 |
| W6 | 🧠 Prompt 温度调整 | 不同模型不同 temperature 配置 |
| W7 | 📦 安装体验 | 提供一键安装脚本、依赖检查 |
| W8 | 📄 CHANGELOG 规范化 | 每次更新写清楚"前因后果" |

### 🎯 Phase 3: 能力扩展（v0.9.x）
| 周次 | 主题 | 内容 |
|------|------|------|
| W9 | 🔌 插件系统 | Skill 自动发现优化 |
| W10 | 🔄 流式输出 | streaming 支持 |
| W11 | 📡 WebSocket | 网关长连接支持 |
| W12 | 🐳 Docker 部署 | 一键容器化 |

## 稳定性 Checklist

- [ ] 会话恢复不污染上下文
- [ ] LLM 回答与工具结果一致性验证
- [ ] 所有异常有明确错误信息
- [ ] 工具超时有降级路径
- [ ] 配置错误有友好提示
- [ ] 网络故障有重试
- [ ] 模型故障回退到备用模型

## 质量 Checklist

- [x] 核心函数有类型注解
- [x] 关键路径有单元测试（134 个测试，5 个模块）
- [x] 错误信息中英文双语（tools.py / cli.py 已全部英文）
- [ ] commit message 说明前因后果
- [ ] CHANGELOG 每周有实质更新
